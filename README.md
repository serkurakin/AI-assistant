# ИИ агент (RAG + Web Search) для анализа pdf-статей

Интеллектуальный агент для анализа научных статей. Система построена на базе LangChain Agents, что позволяет ИИ не просто отвечать на вопросы, а самостоятельно выбирать инструменты для поиска информации.

## Ключевые особенности

- **Agentic Workflow**: Использование `AgentExecutor`.
- **Гибридный поиск**: Агент приоритизирует поиск в локальной базе знаний (RAG), а при отсутствии данных обращается к поиску в реальном времени (Web Search).
- **Метаданные и цитирование**: Система автоматически извлекает названия PDF-файлов и расставляет ссылки вида `[название_файла.pdf]` после каждого значимого утверждения. В конце своего ответа агент использует еще один инструмент - приводится список цитируемой литературы (в виде "Список литературы: 1..., 2... и т.д.")
- **Очистка текста**: Скрипт индексации очищает текст статей и автоматически отсекает раздел "Список литературы" ("References"), чтобы база данных содержала только смысловой текст.
- Бот запоминает историю последних сообщений (до 3 штук) и способен отвечать в Telegram на русском языке в формате диалога.

## Стек технологий

*   **LLM**: GPT-4o-mini (через OpenRouter)
*   **Framework**: LangChain (Core, Classic, Community)
*   **RAG/Vector DB**: ChromaDB (Persistent Storage)
*   **Embeddings**: Sentence-Transformers (paraphrase-multilingual-MiniLM-L12-v2)
*   **Search API**: Tavily Search
*   **Telegram API**: pyTelegramBotAPI

## Структура

*   `bot_v3.py` — Основной файл ИИ-агента и Telegram-бота.
*   `rag_v3.py` — Скрипт для первичной обработки PDF и создания векторной базы.
*   `articles/` — Папка для размещения статей (PDF).
*   `chroma_db/` — Локальное хранилище векторных представлений (создается автоматически).

## Инструкция по запуску

### 1. Подготовка окружения
Установите необходимые зависимости:
```bash
python -m venv .venv
.\.venv\Scripts\activate
pip install -r requirements.txt
```
### 2. Подготовка папок и данных
1. Создайте в корне проекта папку `articles/`.
2. Поместите в неё ваши статьи в формате PDF.

### 3. Настройка переменных окружения (.env)
Создайте файл `.env` и добавьте в него ваши ключи API:

```bash
BOT_TOKEN=ваш_токен_телеграм
OPENROUTER_API_KEY=ваш_ключ_openrouter
TAVILY_API_KEY=ваш_ключ_tavily
```

### 4. Создание векторной базы знаний (RAG)
Перед запуском бота необходимо индексировать статьи. Запустите скрипт и дождитесь его выполнения:
```bash
python rag_v3.py
```

### 5. Запуск ИИ-агента
После того как база данных успешно создана, запустите основного бота:
```bash
python bot_v3.py
```
