# Интеллектуальный ассистент для углубленного анализа научной литературы по естественным наукам

Интеллектуальный агент для анализа англоязычной (и русскоязычной) научной литературы по физике, биофизике, химии и молекулярной биологии. Система построена на базе LangChain Agents, что позволяет ИИ автономно выбирать оптимальный инструмент для решения задачи: локальную базу знаний (RAG), специализированные научные базы или глобальный поиск в сети.


## Ключевые особенности

- **Agentic Workflow**: Использование `AgentExecutor` для автономного выбора инструментов поиска.
- **Многоуровневый поиск (Multi-step Search)**: Агент применяет многоуровневую стратегию:
    1. Приоритетный поиск в локальной базе знаний (**RAG**) на основе статей в формате PDF.
    2. Поиск в рецензируемых базах (**Semantic Scholar**) с Open Access и препринтах (**ArXiv**).
    3. Обращение к поиску в реальном времени (**Web Search**) через Tavily API при дефиците данных.
- **Гибридный поиск по локальной БД**: Объединение двух алгоритмов поиска для максимальной точности:
    1. Векторный поиск: Поиск по смыслу и семантическим связям (используется для сложных запросов).
    2. BM25 (Поиск по ключевым словам): Поиск по точным совпадениям слов (важен для поиска специфических терминов, аббревиатур и имен файлов).
    3. Весовое объединение результатов обоих методов (можно менять веса)
- **Автоматическое цитирование и метаданные из локальной БД**:
    1. Система динамически извлекает названия PDF-файлов из метаданных локальной БД.
    2. Агент расставляет ссылки на первоисточники вида [название_файла.pdf] после каждого значимого утверждения в своем ответе.
    3. Реализован отдельный инструмент для формирования итогового списка литературы, на который ссылался бот по ходу своего ответа.
- **Smart PDF Parsing (Docling)**: используется нейросетевой парсер **IBM Docling**. Это позволяет:
    1. Корректно извлекать сложные таблицы в формате Markdown.
    2. Распознавать структуру документа (заголовки, списки, подписи).
    3. Сохранять логическую связность двухколоночных статей.
- **Очистка текста**: Скрипт индексации очищает текст статей для RAG и автоматически отсекает раздел "Список литературы" ("References"), чтобы база данных содержала только смысловой текст статьи, исключая попадание библиографического «шума» в векторное пространство.
- **Умное резюме**: При индексации агент автоматически создает краткое резюме (Summary) для каждой статьи из векторной БД.
    1. Программа находит разделы *Аннотация (Abstract)*, *Материалы и Методы (Materials and Methods)* и *Заключение (Conclusion)* и анализирует их.
    2. Это позволяет мгновенно отвечать на вопросы: "О чем эта статья?", "Какие методы исследования в этой статье?", если спросить о статье, упомянув название соответствующего файла.
- **Контекстная память**: Бот запоминает историю последних сообщений (до 5 штук) и способен отвечать в Telegram на русском языке в формате диалога.

## Безопасность и устойчивость

Учтены стандарты **OWASP Top 10 for LLM**:
- **Защита от оффтопа**: Фильтрация тем (отказ обсуждать политику, спорные территориальные или исторические темы, бытовые вопросы, экстремистские материалы).
- **Prompt Injection (LLM01)**: Использование системных инструкций для предотвращения смены роли.
- **Excessive Agency (LLM08)**: Исключается риск удаления или модификации данных.

## Стек технологий

*   **LLM**: GPT-4o-mini / Gemini 2.0 Flash или другие модели (через OpenRouter)
*   **Framework**: LangChain (Core, Classic, Community)
*   **RAG/Vector DB**: ChromaDB (Persistent Storage)
    **BM25**: Keyword Search
*   **PDF Engine**: **Docling** (Layout Analysis & OCR).
*   **Embeddings**: Sentence-Transformers (paraphrase-multilingual-MiniLM-L12-v2)
*   **Scientific APIs**: Semantic Scholar API, ArXiv API
*   **Search API**: Tavily Search
*   **Telegram API**: pyTelegramBotAPI

## Структура

*   `bot_v3.py` — Основной файл ИИ-агента и Telegram-бота.
*   `rag_v3.py` — Скрипт для первичной обработки PDF и создания векторной базы.
*   `requirements.txt` — Информация о версиях библиотек.
*   `articles/` — Папка для размещения статей (PDF).
*   `chroma_db/` — Локальное хранилище векторных представлений (создается автоматически).

## Инструкция по запуску

### 1. Подготовка окружения
Установите необходимые зависимости:
```bash
python -m venv .venv
.\.venv\Scripts\activate
pip install -r requirements.txt
```
### 2. Подготовка папок и данных
1. Создайте в корне проекта папку `articles/`.
2. Поместите в неё ваши статьи в формате PDF.

### 3. Настройка переменных окружения (.env)
Создайте файл `.env` и добавьте в него ваши ключи API:

```bash
BOT_TOKEN=ваш_токен_телеграм
OPENROUTER_API_KEY=ваш_ключ_openrouter
TAVILY_API_KEY=ваш_ключ_tavily
```

### 4. Создание векторной базы знаний (RAG)
Перед запуском бота необходимо индексировать статьи. В начале скрипта rag_v3.py (в переменной pdf_folder) пропишите путь к своим статьям.
Запустите скрипт и дождитесь его выполнения:
```bash
python rag_v3.py
```

### 5. Запуск ИИ-агента
После того как база данных успешно создана, запустите основного бота:
```bash
python bot_v3.py
```
